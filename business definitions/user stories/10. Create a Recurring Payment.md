# US-010: Create a Recurring Payment

## Title

Create a recurring payment

## User Story

As a family member, I want to create a recurring payment, so that the family can track upcoming financial obligations and avoid missing payments.

## Acceptance Criteria

* An authenticated family member can create a recurring payment inside a family.
* The recurring payment must have a name.
* The recurring payment must have an expected amount.
* The recurring payment must have a frequency.
* The recurring payment must have a next due date.
* The recurring payment may have a suggested account.
* The recurring payment may have a suggested category.
* The recurring payment can be active or inactive.
* The system should show the recurring payment as an upcoming obligation.
* The recurring payment does not automatically create an expense unless the user confirms or registers the payment.

## Notes

Examples:

```
Name: Internet
Expected Amount: $120,000
Frequency: Monthly
Next Due Date: 2026-06-15
Suggested Account: Bancolombia Savings
Suggested Category: Utilities
```
