# UF-004: Create an Account

## Related User Story

US-004: Create an Account

## Goal

Allow a family member to create a real financial account inside a family.

## Main Flow

1. The family member opens the accounts section.
2. The system shows the list of existing accounts.
3. The family member selects "Create account".
4. The system shows the account creation form.
5. The family member enters the account name.
6. The family member selects the account type.
7. The family member optionally enters an initial balance.
8. The family member submits the form.
9. The system validates the account information.
10. The system creates the account inside the selected family.
11. The system shows the new account in the accounts list.

## Alternative Flows

### Alternative Flow 1: Missing account name

1. The family member submits the form without an account name.
2. The system does not create the account.
3. The system shows a validation message.
4. The family member enters a valid account name.

### Alternative Flow 2: Missing account type

1. The family member submits the form without selecting an account type.
2. The system does not create the account.
3. The system shows a validation message.
4. The family member selects a valid account type.

### Alternative Flow 3: Invalid initial balance

1. The family member enters an invalid initial balance.
2. The system does not create the account.
3. The system shows a validation message.
4. The family member corrects the initial balance.

## Postconditions

* A new account exists inside the selected family.
* The account can be used for income, expenses, and transfers.
* The account balance starts from the defined initial balance or zero.
