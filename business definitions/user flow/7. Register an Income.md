# UF-007: Register an Income

## Related User Story

US-007: Register an Income

## Goal

Allow a family member to register income and update the selected account balance.

## Main Flow

1. The family member selects "Add income".
2. The system shows the income form.
3. The family member enters the income amount.
4. The family member selects the account that receives the income.
5. The family member selects the category.
6. If the category has active subcategories, the system shows a required subcategory selector.
7. The family member selects the subcategory when required.
8. The family member optionally enters a description.
9. The family member optionally selects a transaction date.
10. If no date is selected, the system uses the current date.
11. The family member submits the income.
12. The system validates the income information.
13. The system creates the income transaction.
14. The system increases the selected account balance.
15. The system updates reports and dashboard calculations.
16. The system confirms that the income was registered.

## Alternative Flows

### Alternative Flow 1: Missing amount

1. The family member submits the form without an amount.
2. The system does not create the income.
3. The system shows a validation message.
4. The family member enters a valid amount.

### Alternative Flow 2: Missing account

1. The family member submits the form without selecting an account.
2. The system does not create the income.
3. The system shows a validation message.
4. The family member selects an account.

### Alternative Flow 3: Missing category

1. The family member submits the form without selecting a category.
2. The system does not create the income.
3. The system shows a validation message.
4. The family member selects a category.

### Alternative Flow 4: Missing required subcategory

1. The family member selects a category that has active subcategories.
2. The family member submits the form without selecting a subcategory.
3. The system does not create the income.
4. The system shows a validation message.
5. The family member selects an active subcategory.

## Postconditions

* An income transaction is created.
* The selected account balance is increased.
* If a subcategory was required, the income stores the selected subcategory and the parent category.
* The income is available for reports and dashboard calculations.
