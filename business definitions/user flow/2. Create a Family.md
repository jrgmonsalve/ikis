# UF-002: Create a Family

## Related User Story

US-002: Create a Family

## Goal

Allow an authenticated user to create a family workspace for managing personal or shared finances.

## Main Flow

1. The user signs in successfully.
2. The system detects that the user does not belong to any family.
3. The system redirects the user to the "Create Family" screen.
4. The user enters the family name.
5. The user submits the form.
6. The system validates the family name.
7. The system creates the family.
7a. If an equivalent family creation request was already processed for the same user and normalized family name, the system reuses the existing family instead of creating a duplicate.
8. The system assigns the user as the family owner.
9. The system redirects the user to the family dashboard.

## Alternative Flows

### Alternative Flow 1: Missing family name

1. The user submits the form without a family name.
2. The system does not create the family.
3. The system shows a validation message.
4. The user enters a valid family name.
5. The user submits the form again.

### Alternative Flow 2: Family creation fails

1. The user submits a valid family name.
2. The system fails to create the family.
3. The system shows an error message.
4. The user can try again.

## Postconditions

* A family is created.
* The authenticated user becomes the family owner.
* The user can start creating accounts, categories, transactions, budgets, and recurring payments.
