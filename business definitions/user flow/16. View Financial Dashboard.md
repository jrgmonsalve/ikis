# UF-016: View Financial Dashboard

## Related User Story

US-016: View Financial Dashboard

## Goal

Allow a family member to quickly understand the family’s financial status.

## Main Flow

1. The family member opens the selected family dashboard.
2. The system loads the default dashboard period.
3. The system calculates total available balance.
4. The system calculates total income for the selected period.
5. The system calculates total expenses for the selected period.
6. The system loads budget progress summary.
7. The system loads upcoming recurring payments.
8. The system loads expenses by category.
9. The system shows the dashboard summary.
10. The family member can change the dashboard period.
11. If the period changes, the system recalculates the dashboard data.

## Alternative Flows

### Alternative Flow 1: No transactions exist

1. The family member opens the dashboard.
2. The system detects that there are no transactions.
3. The system shows balances based on existing accounts.
4. The system shows empty states for income, expenses, and category reports.

### Alternative Flow 2: No accounts exist

1. The family member opens the dashboard.
2. The system detects that no accounts exist.
3. The system shows an onboarding message.
4. The system suggests creating the first account.

### Alternative Flow 3: Dashboard calculation fails

1. The system cannot calculate dashboard data.
2. The system shows an error message.
3. The user can retry later.

## Postconditions

* The family member can see a financial summary for the selected family.
* The dashboard shows balances, income, expenses, budget progress, recurring payments, and
expenses by category.
* The dashboard data respects the selected period and selected family.
