# UF-011: Mark a Recurring Payment as Paid

## Related User Story

US-011: Mark a Recurring Payment as Paid

## Goal

Allow a family member to register a recurring payment as paid and create the related expense transaction.

## Main Flow

1. The family member opens the recurring payments section.
2. The system shows pending recurring payments.
3. The family member selects a recurring payment.
4. The family member selects "Mark as paid".
5. The system shows the payment confirmation form.
6. The system preloads the expected amount.
7. The system preloads the suggested account if available.
8. The system preloads the suggested category if available.
9. The family member confirms or edits the payment amount.
10. The family member confirms or selects the account.
11. The family member confirms or selects the category.
12. The family member optionally selects the payment date.
13. If no payment date is selected, the system uses the current date.
14. The family member confirms the payment.
15. The system creates an expense transaction.
16. The system reduces the selected account balance.
17. The system marks the recurring payment as paid for the current period.
18. The system calculates the next due date based on the frequency.
19. The system updates reports and budget calculations.
20. The system confirms that the payment was registered.

## Alternative Flows

### Alternative Flow 1: Missing account

1. The family member confirms the payment without selecting an account.
2. The system does not create the expense.
3. The system shows a validation message.
4. The family member selects an account.

### Alternative Flow 2: Missing category

1. The family member confirms the payment without selecting a category.
2. The system does not create the expense.
3. The system shows a validation message.
4. The family member selects a category.

### Alternative Flow 3: Invalid amount

1. The family member enters an invalid amount.
2. The system does not create the expense.
3. The system shows a validation message.
4. The family member enters a valid amount.

## Postconditions

* An expense transaction is created.
* The selected account balance is reduced.
* The recurring payment is marked as paid for the current period.
* The next due date is updated.
* Reports and budgets include the created expense.
