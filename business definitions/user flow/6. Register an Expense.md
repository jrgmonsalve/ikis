# UF-006: Register an Expense

## Related User Story

US-006: Register an Expense

## Goal

Allow a family member to register an expense quickly and update the selected account balance.

## Main Flow

1. The family member selects "Add expense".
2. The system shows the expense form.
3. The family member enters the expense amount.
4. The family member selects the account used for the expense.
5. The family member selects the category.
6. The family member optionally enters a description.
7. The family member optionally selects a transaction date.
8. If no date is selected, the system uses the current date.
9. The family member submits the expense.
10. The system validates the expense information.
11. The system creates the expense transaction.
12. The system reduces the selected account balance.
13. The system updates reports and budget calculations.
14. The system confirms that the expense was registered.

## Alternative Flows

### Alternative Flow 1: Missing amount

1. The family member submits the form without an amount.
2. The system does not create the expense.
3. The system shows a validation message.
4. The family member enters a valid amount.

### Alternative Flow 2: Missing account

1. The family member submits the form without selecting an account.
2. The system does not create the expense.
3. The system shows a validation message.
4. The family member selects an account.

### Alternative Flow 3: Missing category

1. The family member submits the form without selecting a category.
2. The system does not create the expense.
3. The system shows a validation message.
4. The family member selects a category.

## Postconditions

* An expense transaction is created.
* The selected account balance is reduced.
* The expense is available for reports and budget usage calculations.
