# UF-009: Create a Budget

## Related User Story

US-009: Create a Budget

## Goal

Allow a family member to create a budget for a monthly, yearly, or custom period.

## Main Flow

1. The family member opens the budgets section.
2. The system shows the list of existing budgets.
3. The family member selects "Create budget".
4. The system shows the budget creation form.
5. The family member enters the budget name.
6. The family member selects the budget category.
7. The family member enters the planned amount.
8. The family member selects the budget period type.
9. If the period type is monthly, the family member selects a month and year.
10. If the period type is yearly, the family member selects a year.
11. If the period type is custom date range, the family member selects a start date and an end date.
12. The family member submits the form.
13. The system validates the budget information.
14. The system verifies that there is no duplicate budget for the same category and period.
15. The system creates the budget.
16. The system calculates the current spent amount for the selected category and period.
17. The system shows the budget progress.

## Copy Completed Budget Flow

1. An owner or admin opens the budgets section after a budget period has ended.
2. The system shows the action "Copy to next period" for the completed budget.
3. The owner or admin selects the copy action.
4. The system opens the budget form with the name, category, planned amount, currency, and period type from the source budget.
5. For a monthly budget, the system selects the following month.
6. For a yearly budget, the system selects the following year.
7. For a custom budget, the owner or admin selects a new start date and end date.
8. The owner or admin reviews or modifies the copied values and submits the form.
9. The system validates that the category and new period are not duplicated.
10. The system creates a new budget with progress calculated only for the new period.


## Alternative Flows

### Alternative Flow 1: Missing budget name

1. The family member submits the form without a budget name.
2. The system does not create the budget.
3. The system shows a validation message.
4. The family member enters a valid budget name.

### Alternative Flow 2: Missing category

1. The family member submits the form without selecting a category.
2. The system does not create the budget.
3. The system shows a validation message.
4. The family member selects a category.

### Alternative Flow 3: Invalid custom date range

1. The family member selects a custom date range.
2. The start date is after or equal to the end date.
3. The system does not create the budget.
4. The system shows a validation message.
5. The family member corrects the date range.

### Alternative Flow 4: Duplicate budget

1. The family member submits a budget for a category and period that already has an active budget.
2. The system does not create the budget.
3. The system shows a duplicate budget message.

## Postconditions

* A budget is created inside the selected family.
* The budget can calculate spent amount, remaining amount, and percentage used.
* The budget is available in reports and dashboard summaries.
