# UF-015: View Upcoming Recurring Payments

## Related User Story

US-015: View Upcoming Recurring Payments

## Goal

Allow a family member to see upcoming financial obligations.

## Main Flow

1. The family member opens the recurring payments section or dashboard.
2. The system loads active recurring payments for the selected family.
3. The system filters pending recurring payments.
4. The system orders payments by next due date.
5. The system shows the payment name.
6. The system shows the expected amount.
7. The system shows the next due date.
8. The system shows the suggested account if available.
9. The system shows the suggested category if available.

## Alternative Flows

### Alternative Flow 1: No upcoming recurring payments

1. The family member opens the recurring payments section.
2. The system detects that there are no upcoming recurring payments.
3. The system shows an empty state.
4. The system suggests creating a recurring payment.

### Alternative Flow 2: Some recurring payments are inactive

1. The system loads recurring payments.
2. The system excludes inactive recurring payments from the upcoming list.
3. The system shows only active pending payments.

## Postconditions

* The family member can see upcoming recurring payments.
* Paid payments for the current period are not shown as pending.
