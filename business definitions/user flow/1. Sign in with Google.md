# UF-001: Sign in with Google

## Related User Story

US-001: Sign in with Google

## Goal

Allow the user to access the platform using a Google account.

## Main Flow

1. The user opens the application.
2. The system shows the welcome screen.
3. The user selects "Sign in with Google".
4. The system redirects the user to Google authentication.
5. The user selects or enters their Google account.
6. Google validates the user credentials.
7. The system receives the authentication response.
8. The system checks if the user already exists.
9. If the user does not exist, the system creates a new user profile.
10. If the user already exists, the system loads the existing user profile.
11. The system starts the user session.
12. The system redirects the user according to the routing rules after sign in.

## Routing Rules After Sign In

* If the user has no family, the system redirects the user to create a family.
* If the user belongs to one family, the system redirects the user to that family dashboard.
* If the user belongs to multiple families, the system redirects the user to select a family.

## Alternative Flows

### Alternative Flow 1: User cancels Google authentication

1. The user starts Google authentication.
2. The user cancels the authentication process.
3. The system returns the user to the welcome screen.
4. The system shows an authentication cancellation message.

### Alternative Flow 2: Authentication fails

1. The user starts Google authentication.
2. Google fails to authenticate the user.
3. The system returns the user to the welcome screen.
4. The system shows an authentication error message.

## Postconditions

* The user is authenticated.
* The user profile exists in the platform.
* The user session is active.
